# Preset sync: per-record last-write-wins with tombstones

When logged in, user-created presets sync to and from a Supabase `presets` table (one row per preset, `auth.uid() = user_id` RLS), wrapping the existing `presetStorage.ts` localStorage module rather than replacing it — local reads/writes behave exactly as before, and a sync layer (`presetSync.ts`) pushes and pulls on top. Built-in presets are defined in code and never touch Supabase.

Conflict resolution is per-record, not per-account. On login (and whenever a pull runs), the full remote row set — including tombstoned rows — is fetched and merged against local state via `mergePresets`: for a preset both sides know about, whichever copy has the later `updatedAt` wins; a remote tombstone always wins over a local copy, even one edited more recently than the tombstone, because deletion is treated as authoritative over any edit. Presets only one side has ever seen (a fresh local preset that hasn't synced yet, or a preset created on another device) are kept as-is. Any local preset that comes out of the merge ahead of its remote counterpart — including local-only presets — is pushed back up individually via `pushPreset`, so the merge is symmetric across the two sessions that raced.

Mutations push per-record rather than replacing the whole remote set: `savePreset`/`renamePreset` upsert the single changed row (`pushPreset`), and `deletePreset` removes the preset locally but writes a `deleted_at`/`updated_at` tombstone remotely (`pushTombstone`) instead of deleting the row — the row's `name`/`config` are left untouched so the tombstone still satisfies `NOT NULL` and other devices could inspect it if needed. A preset that has never synced does not need to be sent as a tombstone; there was never a remote row to poison.

This further qualifies [ADR-0002](0002-localstorage-only-persistence.md) and [ADR-0004](0004-supabase-auth-scaffolding.md): presets are no longer local-only when a user is logged in. Logged-out behavior, and behavior when Supabase isn't configured for the build, are unchanged — sync is skipped entirely and everything stays local as before.
