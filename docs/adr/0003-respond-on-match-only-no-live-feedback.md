# Respond-on-match-only input, no live feedback by default

Rather than requiring an explicit yes/no response every trial, the app only registers input when the user asserts a match (silence means "no match" implicitly), matching the convention used by established n-back trainers (e.g. Brain Workshop, Jaeggi-style protocols). Feedback on whether a response was correct is withheld until the session-end summary by default, rather than shown live per trial, because live feedback during real working-memory training is generally considered a distraction/crutch rather than a help. Both choices deviate from what a newcomer might assume (explicit per-trial yes/no with immediate feedback) so are recorded here; live feedback remains available as an opt-in toggle for users who want it while learning the mechanics.
